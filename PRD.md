# Trace（心迹）
## 一个以中国地图为入口的极简旅行回忆记录工具

---
## 一、产品概述
### 1.1 产品背景
帮助用户以空间维度记录人生旅程，将分散的照片、视频和文字整合成一张可持续扩展的个人旅行记忆地图。
用户可以：
- 在中国地图上查看已去过的地方
- 逐级进入省份与城市
- 以图片、视频、文字形式记录与浏览每次旅行的精彩瞬间
- 长期沉淀个人回忆
产品整体风格为参考Apple风格极简设计：
- 留白
- 清晰层级
- 轻交互
- 强视觉沉浸感

---
## 二、目标用户
**核心用户**
- 喜欢记录生活的人
- 旅行爱好者
- 摄影爱好者
- 希望沉淀回忆的用户
**使用场景**
- 旅行结束后整理照片
- 回顾过去足迹
- 记录重要旅程
- 收藏人生记忆

---
## 三、核心价值
1. **空间化记录体验**
   用地图替代传统相册，使旅行经历更具故事感。
2. **回忆沉淀**
   长期记录旅行中的瞬间，形成可持续积累的人生轨迹。
3. **极简使用体验**
   减少复杂操作，让用户专注于内容和记录本身。

---

## 五、功能需求
### 5.1 地图首页
**功能描述**
展示中国地图，作为产品主入口。
**功能要求**
- 显示中国省级行政区地图
- 已记录过旅行的省份高亮显示
- hover 显示省份名称
- 点击进入省份页面
**交互逻辑**
默认状态：
- 地图为浅灰色
- 已记录省份为深色
点击省份：
  进入城市列表页
**UI风格**
- 浅色背景
- 细边界
- 柔和高亮色
- 动效轻量

### 5.2 省份页面
**功能描述**
展示该省份下的城市列表。
**功能要求**
- 展示城市列表
- 已记录城市高亮
- 点击进入城市记录页
**布局方式**
卡片式 / 列表式
**示例：**
山东省
青岛   3条记录
济南   2条记录
威海   0条记录

### 5.3 城市旅行记录页
**功能描述**
展示该城市下所有旅行记录。
**展示内容**
每条记录包含：
- 标题
- 图片
- 视频
- 文字描述
- 时间
- 标签（可选）
**布局方式**
卡片流布局
**卡片内容：**
- 封面图片
- 标题
- 日期
- 文字摘要
**点击卡片：**
  进入记录详情页

### 5.4 记录详情页
**内容结构**
顶部：
- 标题
- 城市
- 日期
主体：
- 图片轮播
- 视频播放器
- 文字内容
底部：
- 标签
- 编辑按钮

### 5.5 新增旅行记录
**功能描述**
允许用户新增旅行记录。
**输入字段**
- 省份（支持搜索）
- 城市（支持搜索，可直接选择城市自动匹配省份）
- 标题
- 日期
- 图片上传（最多9张）
- 视频上传（最多2个）
- 旅行描述
**上传限制（MVP）**
- 图片：jpg/png，≤10MB，最多9张
- 视频：mp4，≤50MB，最多2个

### 5.6 编辑与删除记录
支持：
- 编辑文字
- 删除图片
- 删除记录

  → 查看中国地图
  → 点击省份
  → 点击城市
  → 无记录提示

## 八、数据结构设计
### 8.1 TravelRecord
```typescript
export type MediaItem =
  | { type: "image"; url: string; alt?: string }
  | { type: "video"; url: string; poster?: string };

export interface TravelRecord {
  id: string;
  /** 省级拼音 slug，与地图 /province/[name] 一致，如 shandong、shanghai */
  province: string;
  /** 城市拼音 slug，与 areas.json、/city/[name] 一致，如 qingdao */
  city: string;
  title: string;
  description: string;
  media: MediaItem[];
  date: string;
  tags: string[];
}
```

## 十三、产品迭代方向
当满足基础的记录需求后，产品可以新增旅游攻略分享功能。即针对不同的城市，用户可以发布自己的旅游攻略供其他用户参考，用户点进其中一个省份后会有一个板块展示旅游攻略，包含预算、路线、住宿推荐、各种玩法等，每个人可以分析自己的路线，大家可自行搜索，包括具体的图片行程安排。
